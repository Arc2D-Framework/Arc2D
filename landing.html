At 10KB, or roughly 1300 Lines of Code (uncompressed LOC), the Arc Engine spares no expense. We've aimed for ultra efficiency on mobile, desktop and web devices. A low memory (RAM) footprint and small kernel size gives Arc a solid advantage in boot and Time-to-Interactive (TTI) metrics.

SETTING THE PRECEDENCE

The iPhone set the precedence for handheld design. Thin, curved and glassy. Everyone copied it. In similar fashion, Arc sets the precedence for how a framework should be designed, many will copy the craftsman-ship we pioneered.

Arc is a framework managed architecture. Instead of fiddling with specialized API calls, the framework will call the code you write. Write no code and nothing happens. Leverage HTML, CSS and native JavaScript the way you learned it, breaking the barrier to adoption and lenghtty learning curves. The unobtrusive nature of the Arc Engine allows legacy web pages already existing to be progressively enhanced by real components without having to touch the existing HTML even if the existing page is running other apis and frameworks.

SINGLE PAGE ARCHITECTURE

Real world SPA. SEO & Programmer friendly

Even with "SPA" capabilities marketed by the leading "frameworks", confusion and mystery are still ever present. How does it work, what gets compiled, how does one swap views without reloads, how should it be broken up, what about routing, and, where is my SEO? Trade-offs after trade-offs and akward API calls to "Route" between screen views make it a mess. After binge watching videos, reading tutorials and forum posts you achieve SPA. A megalith build with framework-bending rules, a migrane and needless complexity -- with SEO, if it were even possible.

Enter the Arc Engine where SPA was not an after-thought, not some 3rd party NPM plugin, but the core pillars for structuring complex, consumer facing enterprise applications from jump. No routing, wrappers or api calls, in fact you won't be able to distinguish SPA happening in a HelloWorld when compared to a real application -- whether internal or consumer facing. You will not even realize you were building a true SPA architecture, the engine intelligently does it all.

  • Visual Component Engine
    JavaScript components with auto-skin inheritance & specificity protection. No fiddling with .css files, loading order or style clashing.
  • Drivers
    Inverts low-level dependencies; allows drivers to replace sub-systems (i.e: Animation, Templating and Storage). No more vendor lock-in.
  • Namespaces
    Like Java but for JavaScript. Mirrors code to folder structure for supreior organization of large-scale systems.

W3C WEB COMPONENTS

Agnostic HTML Templating

Plug-in your engine of choice (Mustache, Nunjucks, Handlebars & more). No vendor lock-in to technologies like JSX.

Progressive Enhancement

Stealthfully decorate & enhance existing/legacy pages with Arc, 1 component at a time without altering any existing HTML.

2D Game Loop

Time sensitive simulation of physics, collision and AI with a Unity-like fixed update loop: input, update, render cycle.

Data Binding

Data bind a DOMNode on screen to a class member variable in JavaScript for synchronization of values in both directions.
